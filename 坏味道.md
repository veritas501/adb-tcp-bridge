# 坏味道审查与修复记录

审查依据：项目 `AGENTS.md` 中关于职责边界、代码健康度、错误处理、注释/文档和测试验证的要求。

## 已确认并修复

### 1. `hdcserver/backend.go` 文件职责过载

- 文件/符号：原 `src/internal/hdcserver/backend.go`
- 坏味道：单文件超过 1000 行，同时承载 HDC 后端入口、shell 转换、sync 协议、文件传输、channel framing、属性解析等多组独立职责。
- 依据：这些逻辑共享包内协议常量，但变化原因不同；继续堆在一个文件会让 HDC 后端的协议边界和失败模式难以定位，违反“文件接近或超过 500 行时检查职责边界”的要求。
- 修复：按职责拆分为：
  - `backend.go`：后端入口、配置、服务分发。
  - `shell.go`：`shell:` / `exec:` 与 shell v2 转换。
  - `sync.go`：ADB sync 子协议、远端目录/文件元数据解析。
  - `file_transfer.go`：HDC native file send task 和 protobuf-like payload 编码。
  - `channel.go`：HDC handshake、channelConn、HDC frame 编解码。
  - `properties.go`：target list 和设备属性解析。
- 风险等级：中。
- 状态：已修复。

### 2. `reverseManager` 绕过后端抽象

- 文件/符号：`src/internal/bridge/reverse.go`，`runDeviceReverse` / `closeAll`
- 坏味道：`reverse:` 控制命令直接通过 `config.Host.RunService` 访问本机 adb server，而普通 service 已经通过 `DeviceBackend` 抽象。使用 `-backend hdc` 时，该路径会绕过已选择的后端，仍尝试拿 HDC connect key 去访问 adb server。
- 依据：`Config.Backend` 是实际目标传输边界；`Config.Host` 只是 ADB 后端的实现细节。跨过 `DeviceBackend` 会制造第二条不一致的数据路径。
- 修复：`runDeviceReverse` 改为通过 `config.Backend.OpenService(ctx, serial, "reverse:"+command)` 打开控制 service 并读取完整响应。HDC 后端不支持 `reverse:` 时会返回显式错误，而不是隐式走 adb server。
- 覆盖测试：新增 `TestRunDeviceReverseUsesConfiguredBackend`，验证 reverse 控制命令使用配置的后端和 serial。
- 风险等级：高。
- 状态：已修复。

### 3. HDC frame 与 sync payload 缺少长度上限

- 文件/符号：`src/internal/hdcserver/channel.go::readHDCFrame`，`src/internal/hdcserver/sync.go::readSyncRequest`
- 坏味道：读取长度前缀后直接按声明长度分配内存。异常 HDC server 或异常 ADB sync 客户端可以声明超大 payload，导致内存膨胀。
- 依据：`adbwire.ReadPacket` 已有 `MaxPayloadSize` 防护；HDC channel 和 sync 子协议处于同样的不可信 I/O 边界，应保持一致的资源上限。
- 修复：新增 `maxHDCFrameSize` 和 `maxSyncPayloadSize`，当前均为 `16 << 20`；超过上限时返回明确错误。
- 覆盖测试：新增 `TestReadHDCFrameRejectsOversizeFrame` 和 `TestReadSyncRequestRejectsOversizePayload`。
- 风险等级：高。
- 状态：已修复。

### 4. HDC 远端目录解析静默吞错

- 文件/符号：`src/internal/hdcserver/sync.go::remoteList`，`parseRemoteEntry`
- 坏味道：`parseRemoteEntry` 忽略 size/mtime 的解析错误，未知条目类型也被转换成 mode `0`；`remoteList` 遇到解析错误直接 `continue`。结果是目录项可能被静默隐藏或以错误元数据返回。
- 依据：远端目录列表是 `adb pull` / recursive pull 的可观察契约；格式异常应作为协议/兼容性错误暴露，而不是给调用方制造“目录为空或文件缺失”的假象。
- 修复：`parseRemoteEntry` 对 size、mtime 和条目类型返回显式错误；`remoteList` 遇到解析错误立即返回错误，由 sync 层写回 `FAIL`。
- 覆盖测试：新增 `TestParseRemoteEntryRejectsInvalidMetadata`。
- 风险等级：中。
- 状态：已修复。
